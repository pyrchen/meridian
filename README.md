# Meridian

Утренний дайджест значимых новостей мира — **крипта, ИИ, разработка, бизнес** —
плюс две приватные страницы: **арбитражный терминал** и **анализатор сигналов** по
альткойнам.

- Без логина и трекинга. Открыл — читаешь.
- **PWA**: ставится как приложение, последняя лента доступна **офлайн** утром.
- Бесплатный хостинг и автоматика целиком на GitHub (Pages + Actions). Своих
  серверов и платных сервисов нет — «сервер» это cron-раннер, «база данных» это git.

## Как это устроено

Каждые ~30 минут GitHub Actions:

1. **Новости** (`scripts/fetch-news.mjs`) — тянет RSS/Atom проверенных источников
   (`scripts/sources.json`), кластеризует похожие сюжеты (больше независимых
   источников → выше «значимость»), ранжирует, пишет `public/data/news.json`.
2. **Сигналы** (`scripts/gen-signals.mjs`) — тянет свечи топ-100 альткойнов по
   объёму с Binance, считает индикаторы, генерирует/оценивает сигналы, пишет
   `public/data/signals.json`.
3. Собирает сайт (Vite), коммитит снапшоты данных (история + офлайн-фолбэк +
   keepalive cron'а) и публикует на GitHub Pages.

Сбор идёт server-side, поэтому CORS источников не мешает. Фронт — Vite + TypeScript,
без тяжёлых фреймворков. Service worker (vite-plugin-pwa) кэширует оболочку и данные
(stale-while-revalidate) → офлайн-вид мгновенный, обновляется при сети.

### Три страницы

| Страница | Файл | Доступ | Назначение |
|---|---|---|---|
| Новости | `index.html` | публичный | Дайджест, 4 категории, поиск, тема |
| Терминал | `arbitrage.html` | приватный | Арбитраж по криптобиржам |
| Сигналы | `signals.html` | приватный | Анализатор альткойнов (фьючерсы/спот) |

## Локальный запуск

```bash
pnpm install
pnpm icons      # PWA-иконки из public/icons/icon.svg (нужен sharp)
pnpm news       # собрать public/data/news.json
pnpm signals    # собрать public/data/signals.json (тянет свечи с Binance)
pnpm dev        # http://localhost:5173
```

Полная сборка как в проде: `pnpm build:all` → `pnpm preview`.

## Деплой (GitHub Pages)

1. Создать **публичный** репозиторий (public = безлимит минут Actions; приватный
   Pages требует платный план).
2. Settings → Pages → **Source: GitHub Actions**.
3. Запушить в `main`. `.github/workflows/deploy.yml` соберёт и опубликует; дальше
   обновляется по cron (`7,37 * * * *`). Сайт: `https://<user>.github.io/<repo>/`.

База пути — `base` в `vite.config.ts`. Для своего домена / Cloudflare Pages:
`BASE_PATH=/ pnpm build`.

## Доступ к приватным страницам

Замок — **клиентский** (security-by-obscurity): сравнивается SHA-256 фразы с хэшем
в `src/lib/gate.ts`. Не серверная защита (хэш виден в коде), но отсекает случайных;
страницы помечены `noindex`. Один вход открывает обе приватные страницы за сессию.

- Дефолтная фраза: **`dawn-2026`**
- Сменить — положи в `ACCESS_SHA256` (в `src/lib/gate.ts`) хэш своей фразы:

```bash
node -e "crypto.subtle.digest('SHA-256',new TextEncoder().encode('ТВОЯ_ФРАЗА')).then(b=>console.log([...new Uint8Array(b)].map(x=>x.toString(16).padStart(2,'0')).join('')))"
```

## Терминал (арбитраж)

Сравнивает лучшие **bid/ask** по USDT-парам на биржах с открытым браузерным CORS —
**Binance, Bybit, OKX, Bitget, Kraken, Crypto.com, BitMart** (HTX опцией) — и
показывает, где купить дешевле и продать дороже: спред, нетто (минус комиссии),
число бирж. Авто-обновление 20 с.

> KuCoin, Gate.io, MEXC и др. блокируют браузерный CORS → в чистой статике
> недоступны. Чтобы добавить, нужен лёгкий прокси (например Cloudflare Worker).

## Сигналы (анализатор альткойнов)

Среднесрочно-ориентированный технический анализ топ-100 альткойнов на **трёх
горизонтах**:

| Горизонт | Сигнал | Тренд-фильтр | Срок |
|---|---|---|---|
| Скальп | 1h | 4h | часы |
| Средне | 4h | 1d | дни |
| Долго | 1d | 1d | недели |

- **Фьючерсы** — лонг и шорт, все горизонты, с мини-графиком и уровнями.
- **Спот** — только покупки на средне/долгом горизонте (актив под хранение).
- Стратегия: тренд старшего ТФ + EMA50/200, RSI, MACD, ATR, ADX, объём → скор
  0-100; вход/стоп/тейк по ATR, R:R 2.5.
- **Учёт новостей**: упоминания монеты в свежей ленте + грубый тон могут снизить
  или повысить силу сигнала.
- **Учёт BTC**: нисходящий BTC давит лонги альтов, восходящий — шорты.
- **Прогноз срока** до тейка (по ATR/ADX) + фактический срок по закрытым сигналам.
- **Исход** (TP/SL/истёк) считается по свечам; копится винрейт и средний R.

Параметры — в шапке `scripts/gen-signals.mjs`.

> **Не финансовый совет.** Сигналы и арбитраж могут быть убыточными. Спред ≠
> прибыль (комиссии, перевод, ликвидность, делистинг, коллизии тикеров). Риском,
> размером позиции и плечом управляешь сам.

## Структура

```
scripts/fetch-news.mjs   скрапер RSS → ранжирование → news.json
scripts/gen-signals.mjs  движок сигналов → signals.json
scripts/indicators.mjs   технические индикаторы (EMA/RSI/MACD/ATR/ADX/BB)
scripts/sources.json     список новостных источников
src/news/                новостная страница
src/arb/                 терминал (адаптеры бирж + UI)
src/signals/             анализатор сигналов
src/lib/                 общие утилиты + замок доступа
src/styles/              дизайн-токены и стили
.github/workflows/       cron-сбор + деплой
```
