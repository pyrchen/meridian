# Meridian

Утренний дайджест значимых новостей мира — **крипта, ИИ, разработка, бизнес** —
плюс приватный **арбитражный терминал** по криптобиржам.

- Без логина и трекинга. Открыл — читаешь.
- **PWA**: ставится как приложение, последняя лента доступна **офлайн** утром.
- Бесплатный хостинг и автосбор данных целиком на GitHub (Pages + Actions). Своих
  серверов и платных сервисов нет.

## Как это устроено

**«Git scraping».** Раз в ~2 часа GitHub Actions запускает скрапер
(`scripts/fetch-news.mjs`): тянет RSS/Atom десятков проверенных источников
(`scripts/sources.json`), нормализует, **кластеризует похожие сюжеты** (чем больше
независимых источников пишут об одном — тем выше «значимость»), ранжирует по
свежести + авторитету источника + числу источников и пишет `public/data/news.json`.
Тот же прогон собирает сайт (Vite) и публикует на GitHub Pages. Браузер ничего не
парсит — поэтому CORS источников не мешает.

Фронт — Vite + TypeScript, без тяжёлых фреймворков. Service worker (vite-plugin-pwa)
кэширует оболочку и ленту по стратегии stale-while-revalidate → утренний офлайн-вид
мгновенный, обновляется при сети.

### Две страницы

| Страница | Файл | Назначение |
|---|---|---|
| Новости | `index.html` | Публичный дайджест, 4 категории, поиск, тёмная/светлая тема |
| Терминал | `arbitrage.html` | Приватный арбитражный сканер (см. ниже) |

## Локальный запуск

```bash
pnpm install
pnpm icons     # сгенерировать PWA-иконки из public/icons/icon.svg (нужен sharp)
pnpm news      # один раз собрать public/data/news.json
pnpm dev       # http://localhost:5173
```

Сборка как в проде: `pnpm build` (скрапер + Vite) → `pnpm preview`.

## Деплой (GitHub Pages)

1. Создать **публичный** репозиторий `meridian` (public = безлимит минут Actions).
2. Settings → Pages → **Source: GitHub Actions**.
3. Запушить в `main`. `.github/workflows/deploy.yml` соберёт и опубликует; дальше
   обновляется по cron автоматически. Сайт: `https://<user>.github.io/meridian/`.

База пути задаётся `base` в `vite.config.ts` (`/meridian/`). Для своего домена или
Cloudflare Pages — `BASE_PATH=/ pnpm build`.

## Источники новостей

Список — в `scripts/sources.json` (`category`, `name`, `url`, `weight` 1–10, `lang`).
Добавить/убрать источник — отредактируй файл. Мёртвый фид не валит сборку: скрапер
ловит ошибку и пропускает его.

## Терминал (арбитраж)

Сравнивает лучшие **bid/ask** по USDT-парам на биржах с открытым CORS —
**Binance, Bybit, OKX, Bitget, HTX** — и показывает, где купить дешевле и продать
дороже. Поля: спред (до комиссий), нетто (минус две тейкер-комиссии), число бирж.
Авто-обновление каждые 20 с.

> KuCoin, Gate.io, MEXC блокируют браузерный CORS, Coinbase не отдаёт all-tickers с
> bid/ask — в чистой статике они недоступны. Чтобы добавить их, нужен лёгкий прокси
> (например Cloudflare Worker), читающий их API и отдающий с `Access-Control-Allow-Origin`.

### Код доступа

Замок — **клиентский** (security-by-obscurity): сравнивается SHA-256 фразы с хэшем в
`src/arb/main.ts`. Это не серверная защита (хэш виден в коде), но отсекает случайных.
Страница помечена `noindex` и нигде не залинкована явно как приватная.

- Дефолтная фраза: **`dawn-2026`**
- Сменить — положи в `ARB_PASS_SHA256` хэш своей фразы:

```bash
node -e "crypto.subtle.digest('SHA-256',new TextEncoder().encode('ТВОЯ_ФРАЗА')).then(b=>console.log([...new Uint8Array(b)].map(x=>x.toString(16).padStart(2,'0')).join('')))"
```

> **Не финансовый совет.** Спред между биржами ≠ прибыль: комиссии за вывод, время и
> риск перевода, проскальзывание, реальная ликвидность, заморозка ввода/вывода,
> делистинг. Аномально большой спред — почти всегда мёртвая пара.

## Структура

```
scripts/fetch-news.mjs   скрапер RSS → ранжирование → news.json
scripts/sources.json     список источников
scripts/gen-icons.mjs    SVG → PNG иконки
src/news/                новостная страница
src/arb/                 терминал (адаптеры бирж + UI)
src/styles/              дизайн-токены и стили
.github/workflows/       cron-сбор + деплой
```
